<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM-authored body → rendered HTML | Untrusted Markdown/HTML from the agent is converted and emailed |
| LLM-authored subject → SMTP headers | Untrusted subject string becomes an email header |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering/Elevation | render.go markdown→HTML | mitigate | `bluemonday.UGCPolicy().Sanitize()` on the goldmark fragment only (SEC-03); verified by TestRender_SanitizesScript |
| T-01-02 | Tampering | render.go subject handling | mitigate | `stripCRLF(subject)` before building the prefix (SEC-03/D-07); verified by TestRender_StripsCRLF |
| T-01-07 | Tampering | template.HTML injection | mitigate | Inject only the already-sanitized fragment via `template.HTML`; never pass unsanitized agent HTML to template.HTML (Pitfall 2) |
</threat_model>
