Keep `From == GMAIL_USERNAME` so SPF/DMARC stays aligned on blacktoaster.com (Pitfall 5). Never log the app password.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process → Gmail SMTP | Outbound authenticated TLS connection carrying the app password |
| config recipient → envelope To | Fixed recipient becomes the message destination |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Spoofing/Abuse | email.go To field | mitigate | `To` set only from config recipient; Send takes no recipient param (SEC-02) |
| T-01-04 | Spoofing | email.go From field | mitigate | `From == GMAIL_USERNAME` on the DMARC domain so SPF aligns and DMARC p=reject passes (EMAIL-04, Pitfall 5) |
| T-01-05 | Information Disclosure | app password | mitigate | Password passed only to go-mail `WithPassword`; never logged (redaction tooling is Phase 2) |
| T-01-08 | DoS (self) | SMTP failure path | accept | Surface SMTP errors verbatim as tool errors (D-09); no retry loop — dedup/rate-limit is Phase 2 |
</threat_model>
